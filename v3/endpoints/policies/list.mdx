Lists all deployment policies configured for the specified formation.

Returns a paginated collection of policies that define rules and constraints for deployments and resource management. Policies are executed in sequence order during deployments.

<Callout type="warning" title="Feature Flag Required">
This endpoint requires the formations feature flag to be enabled on your account. Contact support@cloud66.com if this feature is not available.
</Callout>

<RelatedModels models={['Policy', 'Formation', 'Stack']} />

**Policy Properties:**
- **name**: Unique policy identifier within the formation
- **sequence**: Execution order (lower numbers execute first)
- **body**: YAML/JSON policy configuration
- **tags**: Categorization labels for organization
- **version**: Policy version for tracking changes

<Callout type="info" title="Content Control">
Use the `full_content=1` query parameter to include complete policy body content. By default, policy bodies may be truncated for performance.
</Callout>

**Use Cases:**
- **Security Policies**: Network policies, security contexts, pod security standards
- **Resource Policies**: Resource limits, quotas, priority classes
- **Deployment Policies**: Deployment strategies, rollback configurations
- **Compliance Policies**: Regulatory compliance and governance rules
