Retrieves detailed information about a specific formation transformation.

Returns comprehensive details about the specified transformation, including the complete rule configuration, version history, and metadata for deployment customization management.

<Callout type="warning" title="Feature Flag Required">
This endpoint requires the formations feature flag to be enabled on your account. Contact support@cloud66.com if this feature is not available.
</Callout>

<RelatedModels models={['Transformation', 'Formation', 'Stack']} />

**Transformation Details:**
- **Rule Configuration**: Complete YAML transformation definition with filters and transforms
- **Sequence Order**: Execution priority during deployments
- **Version Tracking**: Transformation version and modification history
- **Tag Management**: Organization and categorization labels
- **Access Control**: User permissions and modification tracking

<Callout type="info" title="Permission Required">
You must have transformation view rights on the formation to access transformation details.
</Callout>

**Transformation Structure:**
- **Filters**: Define which Kubernetes resources to target
- **Transforms**: Specify modifications to apply to matched resources
- **Operations**: Support add, replace, remove, copy, move, and test operations
- **JSON Patch**: Uses JSON Patch specification for precise modifications

**Common Transformation Operations:**
- **replace**: Replace existing values at specified paths
- **add**: Add new fields or array elements
- **remove**: Delete fields or array elements
- **copy**: Copy values from one path to another
- **move**: Move values between paths
- **test**: Validate values before applying other operations

**Filter Matching:**
- **kind**: Kubernetes resource type (Deployment, Service, etc.)
- **metadata**: Labels, annotations, names, and namespaces
- **spec**: Resource specification fields
- **status**: Resource status fields (read-only)
