Lists all formation transformations configured for the specified formation.

Returns a paginated collection of transformations that modify formations during deployment based on various rules and filters. Transformations are executed in sequence order during deployments.

<Callout type="warning" title="Feature Flag Required">
This endpoint requires the formations feature flag to be enabled on your account. Contact support@cloud66.com if this feature is not available.
</Callout>

<RelatedModels models={['Transformation', 'Formation', 'Stack']} />

**Transformation Properties:**
- **name**: Unique transformation identifier within the formation
- **sequence**: Execution order (lower numbers execute first)
- **body**: YAML transformation rules and filters
- **tags**: Categorization labels for organization
- **version**: Transformation version for tracking changes

<Callout type="info" title="Content Control">
Use the `full_content=1` query parameter to include complete transformation body content. By default, transformation bodies may be truncated for performance.
</Callout>

**Transformation Types:**
- **Environment Configuration**: Environment-specific settings and variables
- **Resource Management**: Resource limits, requests, and optimization
- **Security Hardening**: Security contexts and policy enforcement
- **Service Configuration**: Service mesh and networking modifications
- **Scaling Rules**: Auto-scaling and replica configurations

**Execution Order:**
- Transformations execute in sequence order during deployments
- Lower sequence numbers execute first
- Dependencies between transformations should be considered when ordering
