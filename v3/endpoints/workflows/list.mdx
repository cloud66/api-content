Lists all deployment workflows configured for the specified formation.

Returns a paginated collection of workflows that define automated sequences of deployment and operational tasks. Workflows control how deployments are executed within the formation.

<Callout type="warning" title="Feature Flag Required">
This endpoint requires the formations feature flag to be enabled on your account. Contact support@cloud66.com if this feature is not available.
</Callout>

<RelatedModels models={['Workflow', 'Formation', 'Stack']} />

**Workflow Properties:**
- **name**: Unique workflow identifier within the formation
- **default**: Whether this workflow is the default for the formation
- **body**: YAML workflow configuration defining steps and execution
- **tags**: Categorization labels for organization
- **version**: Workflow version for tracking changes

<Callout type="info" title="Content Control">
Use the `full_content=1` query parameter to include complete workflow body content. By default, workflow bodies may be truncated for performance.
</Callout>

**Workflow Types:**
- **CI/CD Workflows**: Build, test, and deployment automation
- **Maintenance Workflows**: Backup, cleanup, and maintenance tasks
- **Scaling Workflows**: Auto-scaling and resource management
- **Monitoring Workflows**: Health checks and alerting sequences

**Default Workflow:**
- Each formation has one default workflow
- The default workflow is used when no specific workflow is specified
- Only one workflow per formation can be marked as default
