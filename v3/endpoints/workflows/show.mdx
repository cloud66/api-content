Retrieves detailed information about a specific deployment workflow.

Returns comprehensive details about the specified workflow, including the complete step configuration, version history, and metadata for deployment automation management.

<Callout type="warning" title="Feature Flag Required">
This endpoint requires the formations feature flag to be enabled on your account. Contact support@cloud66.com if this feature is not available.
</Callout>

<RelatedModels models={['Workflow', 'Formation', 'Stack']} />

**Workflow Details:**
- **Step Configuration**: Complete YAML workflow definition with all steps
- **Default Status**: Whether this workflow is the formation's default
- **Version Tracking**: Workflow version and modification history
- **Tag Management**: Organization and categorization labels
- **Access Control**: User permissions and modification tracking

<Callout type="info" title="Permission Required">
You must have workflow view rights on the formation to access workflow details.
</Callout>

**Common Step Types:**
- **build_step**: Build application containers and artifacts
- **test_step**: Run test suites and validation
- **security_step**: Security scanning and vulnerability checks
- **deploy_step**: Application deployment with various strategies
- **script_step**: Custom scripts and commands
- **notification_step**: Alert and notification triggers

**Deployment Strategies:**
- **rolling**: Gradual replacement of instances
- **blue_green**: Switch between two identical environments
- **canary**: Gradual traffic shifting to new version
- **recreate**: Stop all instances then create new ones
