# HelmRelease

Helm releases represent installed instances of Helm charts within Kubernetes formations. They manage the lifecycle of applications deployed via Helm package manager, including installation, upgrades, and configuration management.

<Model>
    <ModelProperty name="uid" type="string">
        The unique identifier of the Helm release.
    </ModelProperty>
    <ModelProperty name="display_name" type="string">
        The display name for the Helm release instance.
    </ModelProperty>
    <ModelProperty name="chart_name" type="string">
        The name of the Helm chart being deployed.
    </ModelProperty>
    <ModelProperty name="repository" type="string">
        The URL of the Helm repository containing the chart.
    </ModelProperty>
    <ModelProperty name="version" type="string">
        The version of the Helm chart being deployed.
    </ModelProperty>
    <ModelProperty name="body" type="string">
        The YAML configuration body for the Helm release (included when full_content=true).
    </ModelProperty>
    <ModelProperty name="created_at_iso" type="string">
        The ISO 8601 timestamp when the Helm release was created.
    </ModelProperty>
    <ModelProperty name="updated_at_iso" type="string">
        The ISO 8601 timestamp when the Helm release was last updated.
    </ModelProperty>
</Model>

**Key Features:**
- **Chart Management**: Deploy and manage Helm charts from repositories
- **Version Control**: Specify exact chart versions for reproducible deployments
- **Configuration**: Custom YAML configurations for chart values
- **Lifecycle Management**: Handle installation, upgrades, and rollbacks

**Common Use Cases:**
- **Infrastructure Components**: Ingress controllers, monitoring tools, databases
- **Application Deployments**: Microservices and application stacks
- **Third-party Services**: External tools and services integration
