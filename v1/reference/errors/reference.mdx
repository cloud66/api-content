The Cloud 66 API uses conventional HTTP response codes to indicate success or failure of API requests.

## HTTP Status Codes

| Error Code | Meaning |
|------------|---------|
| 200 | OK -- Request succeeded |
| 400 | Bad Request -- Your request is invalid |
| 401 | Unauthorized -- Your API key is wrong |
| 403 | Forbidden -- The requested resource is forbidden for your access level |
| 404 | Not Found -- The specified resource could not be found |
| 405 | Method Not Allowed -- You tried to access a resource with an invalid method |
| 406 | Not Acceptable -- You requested a format that isn't JSON |
| 410 | Gone -- The requested resource has been removed from our servers |
| 422 | Unprocessable Entity -- Your request contains invalid parameters |
| 429 | Too Many Requests -- You're making too many requests! Slow down! |
| 500 | Internal Server Error -- We had a problem with our server. Try again later |
| 503 | Service Unavailable -- We're temporarily offline for maintenance. Please try again later |

## Error Response Format

When an error occurs, the API returns a JSON response with error details:

```json
{
  "error": "Error message description",
  "details": "Additional error details if available"
}
```
